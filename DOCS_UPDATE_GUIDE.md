# 文档更新指南

本文档说明如何实时更新项目文档，保持文档与开发进度同步。

## 📋 文档结构

项目包含以下主要文档：

1. **README.md** - 项目主文档（最重要）
2. **CHANGELOG.md** - 更新日志
3. **DEVELOPMENT.md** - 开发指南
4. **PROJECT_STRUCTURE.md** - 项目结构文档
5. **todo.md** - 开发任务清单

---

## 🔄 实时更新流程

### 1. 完成功能后更新 README.md

#### 更新开发进度部分

找到 `README.md` 中的 `## 📊 开发进度` 部分，更新对应阶段的完成度：

```markdown
### 第二阶段：核心页面开发 🚧

- [x] 布局组件（Header、Footer、Sidebar） ✅ 已完成
- [x] 首页开发（轮播图、文章列表） ✅ 已完成
- [ ] 文章列表页
- [ ] 文章详情页

**完成度**：50% 🚧  # 更新百分比
```

#### 更新功能特性部分

在 `## ✨ 功能特性` 部分，将完成的功能从未实现移到已实现：

```markdown
### 已实现功能

- [x] 项目基础搭建
- [x] 路由系统配置
- [x] 布局组件（Header、Footer、Sidebar）  # 新增
- [x] 首页开发  # 新增
```

#### 更新最后更新时间

在开发进度部分顶部更新：

```markdown
> 📅 **最后更新**：2026-01-29  # 更新日期
```

### 2. 发布版本时更新 CHANGELOG.md

#### 创建新版本记录

在 `CHANGELOG.md` 顶部添加新版本：

```markdown
## [0.2.0] - 2026-01-29

### ✨ 新增

- 布局组件（Header、Footer、Sidebar）
- 首页开发（轮播图、文章列表）
- 文章卡片组件

### 🔧 优化

- 响应式布局优化
- 样式系统改进

### 🐛 修复

- 修复路由跳转问题
- 修复样式冲突问题
```

#### 更新未发布部分

将已完成的功能从未发布移到新版本：

```markdown
## [未发布]

### 计划中
- 文章详情页
- 分类和标签系统
- 搜索功能
```

### 3. 添加新功能时更新 DEVELOPMENT.md

#### 添加代码示例

在 `## 💡 代码示例` 部分添加新的示例：

```markdown
### 使用新功能

```typescript
// 示例代码
```

#### 更新常见问题

在 `## ❓ 常见问题` 部分添加新问题：

```markdown
### Q: 如何使用新功能？

A: 参考 [相关章节](#相关章节) 的说明。
```

### 4. 更新 todo.md

#### 标记完成的任务

将完成的任务标记为完成：

```markdown
- [x] 布局组件（Header、Footer、Sidebar）  # 改为 [x]
```

#### 添加新任务

在相应阶段添加新任务：

```markdown
### 2.3 文章列表页
- [ ] 实现文章列表展示
- [ ] 实现筛选功能
- [ ] 实现分页功能
```

## 📝 更新检查清单

每次完成功能后，检查并更新以下内容：

### ✅ README.md

- [ ] 更新开发进度（完成度百分比）
- [ ] 更新功能特性列表
- [ ] 更新最后更新时间
- [ ] 更新版本号（如果发布新版本）

### ✅ CHANGELOG.md

- [ ] 添加新版本记录（如果发布）
- [ ] 更新未发布部分
- [ ] 记录所有变更（新增、优化、修复、删除）

### ✅ DEVELOPMENT.md

- [ ] 添加新的代码示例（如果需要）
- [ ] 更新常见问题（如果有新问题）
- [ ] 更新开发流程（如果有新流程）

### ✅ todo.md

- [ ] 标记完成的任务
- [ ] 添加新任务（如果需要）
- [ ] 更新任务优先级

## 🔗 文档关联

文档之间的关联关系：

```
README.md (主文档)
├── 引用 CHANGELOG.md (更新日志)
├── 引用 DEVELOPMENT.md (开发指南)
├── 引用 PROJECT_STRUCTURE.md (项目结构)
└── 引用 todo.md (开发任务)

CHANGELOG.md
└── 与 README.md 的更新日志同步

DEVELOPMENT.md
└── 补充 README.md 的开发指南部分
```

---

## 💡 最佳实践

1. **及时更新**：完成功能后立即更新文档
2. **保持同步**：确保所有文档信息一致
3. **详细记录**：记录所有重要变更
4. **版本管理**：重要更新时创建新版本
5. **定期检查**：定期检查文档是否需要更新
